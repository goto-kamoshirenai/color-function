# Agents.md

このファイルは、AI エージェント（Claude Code など）が本プロジェクトで作業を行う際に従うべきルールを定義します。

## 自動コミットの方針

AI は作業を行った際、**適切な粒度で自動的にコミットを作成する**こと。ただし **push は絶対に自動実行しない**（push はユーザーの明示的な指示がある場合のみ実行する）。

### コミットの粒度

- **1 コミット = 1 つの論理的な変更単位** とする。複数の無関係な変更を 1 コミットにまとめない。
- 以下のような切れ目で分割する:
  - 機能追加と、その後のバグ修正
  - 依存関係の更新と、それに伴うコード修正
  - 設定ファイルの変更と、アプリケーションコードの変更
  - 複数機能を追加する場合は、機能ごとに分割
- ただし、過度に細かく分けない（例: 同じ機能のための複数ファイル変更は 1 コミットにまとめる）。

### コミット対象に含めないもの

- `.env*` などの秘密情報を含むファイル
- `node_modules/`、`.next/` などの生成物
- 個人環境依存のファイル（`.vscode/settings.json` など、チームで共有しないもの）
- `git add -A` や `git add .` ではなく、**変更したファイルを明示的に指定**してステージングする

### コミット前の確認

- `git status` で意図しないファイルが含まれていないか確認
- `git diff --cached` でステージング内容を確認
- 大規模変更の場合は、コミット前にビルド（`pnpm build`）が通ることを確認する

## コミットメッセージ規則

本プロジェクトの過去のコミット履歴に基づく規則です。

### 基本フォーマット

```
<PREFIX> <日本語の説明>
```

- プレフィックスは**大文字**
- プレフィックスと説明の間は**半角スペース 1 つ**（コロンは付けない）
- 説明は**日本語**で簡潔に（体言止めまたは連体形を基本とする）

### プレフィックス一覧

| プレフィックス | 用途 | 例 |
|---|---|---|
| `ADD` | 新機能・新規ファイル・新規依存の追加 | `ADD テンプレートパネル追加` |
| `FIX` | バグ修正・UI 調整・不要コード削除・脆弱性対応 | `FIX console.log削除` |

### 実例（プロジェクト履歴より）

```
ADD テンプレートパネル追加
ADD 相対輝度機能追加
ADD プレビューカード追加
ADD helpModal追加
ADD CIEDE分析機能追加
ADD 色の展開機能追加
FIX 脆弱性対応
FIX console.log削除
FIX sidebarのスタイルの調整
FIX z-index調整
```

### 分類に迷う場合

- **機能の追加・拡張** → `ADD`
- **既存機能の修正・調整・削除・セキュリティ対応** → `FIX`
- 依存関係のアップグレードやリファクタリングも、実態に合わせて `ADD` または `FIX` を選ぶ

### 禁止事項

- 英語の Conventional Commits 形式（`feat:`, `fix:`, `chore:` など）は使わない
- プレフィックスなしのメッセージは避ける（過去に数件あるが、規則としては必須）
- `Co-Authored-By` 等のフッター追記は、ユーザーが明示的に求めた場合のみ
