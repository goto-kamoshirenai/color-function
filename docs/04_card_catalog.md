# 04. カードカタログ（可視化軸の網羅リスト）

> ステータス: **全カード実装済み**（2026-06-12。各行は単独または統合カードとして実装。
> 用語集は /learn、値コピー・共有リンクは横断機能として実現） / 最終更新: 2026-06-12
> 関連: [コンセプト](./01_concept.md) / [制約・前提](./02_constraints.md) / [UI骨格](./03_ui_skeleton.md)

コンセプト「考え得る限り可視化する」に沿って、**実装しうるカードを網羅的に列挙**する。
ここは"何が候補になりうるか"の発散リストであり、確定仕様ではない。
MVP選定（§末尾）と各カードの計算仕様は後続で精緻化する。

---

## 凡例

各カードに以下のタグを付す。

- **単位**: そのカードが意味を持つ基本単位 … `単`=単色 / `ペ`=ペア / `パ`=パレット
- **観点**: [UI骨格 §3](./03_ui_skeleton.md) の2軸のどちらに属するか … `検`=検証 / `設`=設計
- **依存**: 必要な静的アセット（[制約 §2.2](./02_constraints.md)）… `辞`=色名辞書 / `WCAG`=基準値 / `調`=調和ルール/テンプレ / `覚`=色覚プロファイル / `—`=計算のみ
- **★**: MVP候補（最初に作る最有力。旧V3実装実績 or 価値が高く依存が軽いもの）

> 多くの指標はクライアント計算のみで完結する（[制約 §4](./02_constraints.md)）。`依存`が`—`のものは静的アセット不要。

## MVP 候補のまとめ（★を観点別に整理）

最初に作る輪郭。**「依存が軽く・価値が高く・旧V3で実証済み」**を基準に選んだ初期案。

### 検証（Validation）

- 単色: RGB/HEX・HSL・HSV・OKLCH / 相対輝度 / 対白・対黒コントラスト / 最寄り色名 / 色相環
- ペア: **WCAGコントラスト比** / CIEDE2000色差 / テキスト可読性プレビュー / 色覚シミュレーション
- パレット: コントラスト網羅マトリクス / 色差マトリクス / エントロピー / 色相分布 / SVG・UIプレビュー / 色覚識別性

### 設計（Design）

- 単色: 調和スキーム生成 / トーン展開 / 明度・彩度バリエーション
- ペア: 相手色提案 / 2色間グラデーション
- パレット: 配色テンプレート適用

### 横断

- 値コピー / 共有リンク / 指標ヘルプ

## 未決事項（次に詰める）

1. ★候補のうち、**第1スプリントに載せる最小セット**の確定（おそらく単色＋ペアの検証から）。
2. 各カードの **計算仕様**（使用色空間・丸め・基準値の出典）を指標ごとに文書化。
3. 静的アセットの **スキーマ定義**（色名辞書・WCAG基準・調和ルール・色覚プロファイル）。
4. カードの **共通インターフェース**（[UI骨格 §5](./03_ui_skeleton.md) の `appliesTo` と描画契約）の確定。
