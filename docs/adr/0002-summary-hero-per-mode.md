# 0002. 各モードにサマリーヒーローを置く

- ステータス: 採択
- 日付: 2026-06-13

## 文脈

ペア×検証だけが、先頭に「ヒーローカード」（コーナーブラケット＋大きな見出し数値＋
判定バッジ＋プレビュー帯, [docs adr 0001 とは別の WCAG カード]）を持ち、画面に強い
焦点と階層を与えていた。一方、単色×検証・パレット×検証・設計は、部品レベルの意匠
（`CardFrame`・モノ数値・accent タグ・gap-px グリッド）は揃っているものの、均一な
重みのカードの羅列で主役（焦点）がなく、相対的に平坦で「一貫性がない」印象だった。

部品は揃っているので、不足しているのは **画面レベルの階層＝各モードの主役** だった。

## 決定

各モードの先頭に、そのモードを一目で要約する **サマリーヒーロー** を 1 枚置く。
`CardFrame` の `hero`（ブラケット枠）を用い、ペア×検証と同じ「主役感」を与える。
ヒーローの中身はモードごとに最適化する（単一の大数値が無いモードもあるため）:

- **単色×検証 — 色値（アイデンティティ型）**: 大スウォッチ＋大きな色値＋最寄り色名＋
  RGB/HSL＋相対輝度・対白/対黒コントラスト。従来の「色値」「最寄り色名」カードを
  ヒーローに吸収する。
- **パレット×検証 — パレット概要**: 大スウォッチ列＋健全性の要約（色数・最小コント
  ラスト比・紛らわしいペア数 ΔE00<10・色相エントロピー・グレースケール耐性）。
  詳細カード（マトリクス等）は下に残す。
- **設計 — ベース配色**: ベース色を主役にし、代表的な調和（補色・類似色）をクリックで
  追加できる。詳細な調和スキーム生成カードは下に残す。

## 代替案

- **先頭カードを単にヒーロー枠化** — 既存の先頭カードに `hero` を付けるだけ。低コスト
  だが、単色の「色値」やパレットの「マトリクス」は要約ではなく、モードの全体像を
  伝える主役にはなりにくい。
- **細部（余白・タイポ）だけ磨く** — 一貫性は上がるが、平坦さ（焦点の不在）は解消
  しない。

## 帰結

- 新しいモードやカードを足す人は、**そのモードに主役（サマリーヒーロー）があるか**を
  基準にする。先頭は要約、以降は詳細、という階層を保つ。
- ヒーローは新規カードなので registry / レイアウト / i18n / ヘルプ(HELP) を追加する。
- 単色ヒーローが「色値」「最寄り色名」を吸収しても、両カードの registry 登録は残す
  （/learn の指標別リファレンスの出典として使うため）。代わりにマストヘッドの
  「CARDS — N」は **実際に配置（描画）されるカード数** で数える（registry 長ではない）。
- サマリーヒーローは単一指標ではないため参考資料を持たないことがある（例: パレット
  概要 `overview`）。references のレジストリ網羅テストからは除外する。
- 今回はヒーロー新設まで。スカスカ/不均衡カードの整理と細部の磨きは次ラウンド。
