# 02. 制約・前提

> ステータス: ドラフト / 最終更新: 2026-06-10
> 関連: [コンセプト](./01_concept.md)

コンセプトからは切り離した、**実装上の境界条件**をここに集約する。
基本設計はこの制約の枠内で行う。

---

## 1. 技術スタック

| 項目 | 採用 | 補足 |
|------|------|------|
| フレームワーク | Next.js | App Router 前提（既存資産も Next.js） |
| 言語 | TypeScript | |
| 実行形態 | フロントエンド + Next.js バックエンド | API Routes / Server Components を利用可 |
| データベース | **持たない** | 永続化のための DB は導入しない |

---

## 2. データの持ち方

### 2.1 原則：DB なし
- ユーザーが作った配色・分析結果を**サーバ側に永続保存しない**。
- これは制約であると同時に、**「色データをサーバに預けない」というプライバシー上の利点**でもある。

### 2.2 静的データアセット（バックエンド側に持つ）
DB は持たないが、**バックエンドに静的な参照データ（JSON 等）を持つことは可能**。
本アプリは以下を「根拠データ」として内蔵する（優先実装対象）。

| アセット | 内容 | 用途 |
|---------|------|------|
| 色名辞書 | CSS named colors / JIS 慣用色名 / 和色 等の「名前 ⇔ 値」 | 色への意味づけ・検索・表示 |
| WCAG / 基準値 | コントラスト閾値・AA/AAA 判定基準（将来 APCA も検討） | 検証ロジックの根拠 |
| 調和ルール / テンプレ | 補色・トライアド・類似色などの調和定義、用途別配色テンプレート | 設計支援の素 |
| 色覚多様性プロファイル | P/D/T 型のシミュレーション用変換パラメータ | 色覚シミュレーション |

- これらは**読み取り専用の静的アセット**として配置し、ビルド時または API 経由で配信する。
- 配置方法（`public/` 静的配信／API Routes／バンドル同梱）は基本設計で決定する。

### 2.3 ユーザーの作業状態の持ち方
DB がないため、ユーザーの「いま編集中の配色」は以下で扱う（基本設計で確定）。

- **URL 状態**: 配色を URL クエリ/ハッシュにエンコードし、共有・復元可能にする（DB レスでの共有手段）。
- **ブラウザローカル**: `localStorage` 等での一時保存・履歴（任意機能）。

> URL 状態は、後述するエクスポート Phase の「共有リンク」とも整合する。

---

## 3. エクスポート方針（段階導入）

成果物の受け渡しは段階的に拡張する。

| Phase | 範囲 | 内容 |
|-------|------|------|
| **Phase 1（初期）** | 閲覧・コピー中心 | 画面上で可視化し、HEX/HSL 等を個別コピー。共有は URL 状態で実現。 |
| **Phase 2（将来）** | デザイントークン出力 | CSS 変数 / Tailwind config / JSON トークン等、開発に直接渡せる形式で書き出す。 |

- Phase 1 を軽く保ち、MVP の立ち上げを優先する。
- Phase 2 のトークン設計は、Phase 1 のデータモデル（色・パレットの内部表現）と矛盾しないように先回りして考慮する。

---

## 4. 非機能・前提

- **プライバシー**: ユーザーの配色データはサーバに保存しない（DB なしの帰結）。送信が必要な処理は最小化する。
- **オフライン耐性**: 静的アセット中心のため、可視化・計算の多くはクライアント側で完結しうる（将来的な PWA 化の余地）。
- **計算の所在**: コントラスト比・色差・色覚変換などの計算は、原則クライアント側で実行可能（サーバ往復を必須にしない）。
- **アクセシビリティ**: アクセシビリティを評価するアプリ自身が、UI としてアクセシブルであること。

---

## 5. この制約が設計に与える含意

- 「保存＝URL/ローカル」になるため、**配色の内部表現は URL にエンコード可能なほどコンパクト**であるべき。
- 静的アセットは増えうる（色名辞書は特に大きくなりうる）ため、**遅延読み込み／分割配信**を設計段階で考慮する。
- DB がない＝**マイグレーションや整合性管理の負債がない**。その分、アセットの**バージョン管理（データ定義のスキーマ）**を文書側で担保する。
