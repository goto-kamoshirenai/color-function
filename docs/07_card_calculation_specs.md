# 07. MVPカード計算仕様（スプリント1）

> ステータス: ドラフト / 最終更新: 2026-06-10
> 関連: [データモデルとカード契約](./05_data_model_and_card_contract.md) / [静的アセット](./06_static_assets_schema.md) / [カードカタログ](./04_card_catalog.md)

[05 §8](./05_data_model_and_card_contract.md) のスプリント1カードについて、`core/color/*` が実装できる粒度で
**入力・式・使用色空間・丸め・出典**を確定する。式は標準準拠であり、§10の参照値でテスト可能にする。

対象カード:
- 単色: HEX/RGB・HSL・HSV・相対輝度・対白/黒コントラスト
- ペア: WCAGコントラスト比・CIEDE2000色差・テキスト可読性プレビュー

---

## 1. 共通規約

| 項目 | 規約 |
|------|------|
| 内部精度 | すべて倍精度浮動小数で計算。**途中で丸めない**。丸めは表示直前のみ |
| 色域 | sRGB 固定（[05 §1](./05_data_model_and_card_contract.md)）。基準白色点 D65 |
| RGB 正規化 | 計算は 0–1 正規化（`c01 = c255 / 255`）。保存・HEX のみ 8bit |
| 角度 | 度数法。色相は `[0, 360)` に正規化 |
| 無彩色 | 彩度0（R=G=B）のとき色相は不定。状態上は `0`、表示は「—」 |
| アルファ | 指標計算は原則不透明色で行う。合成が要る箇所は明記 |

### 1.1 sRGB ⇄ 線形RGB（全計算の基礎）
ガンマの解除/付与。`C` は 0–1 正規化チャンネル。

```
線形化（gamma decode）:
  C_lin = C/12.92                         (C ≤ 0.04045)
        = ((C + 0.055)/1.055)^2.4         (C > 0.04045)

ガンマ付与（gamma encode）:
  C = 12.92 · C_lin                       (C_lin ≤ 0.0031308)
    = 1.055 · C_lin^(1/2.4) − 0.055       (C_lin > 0.0031308)
```
出典: IEC 61966-2-1（sRGB）。

## 2. HEX / RGB カード（正準表現）

**目的**: 正準値の表示とコピー（カタログ A / L）。

- **入力パース**: `#rgb` `#rgba` `#rrggbb` `#rrggbbaa` を受理。短縮形は各桁を複製（`#abc`→`#aabbcc`）。
- **出力**: 小文字 `#rrggbb`、アルファ<1 のとき `#rrggbbaa`。RGB は整数 0–255。
- **丸め**: 8bit 量子化は `round(c01 · 255)`。
- **エッジ**: 不正文字列は無効として弾く（パース失敗を `apply` に渡さない）。

---

## 3. HSL カード

**入力**: sRGB（0–1）。**出力**: H[0,360)、S/L[0,100]%。

```
max = max(R,G,B), min = min(R,G,B), Δ = max − min
L = (max + min) / 2
S = 0                       (Δ = 0)
  = Δ / (1 − |2L − 1|)      (otherwise)
H = 60 · ( ((G−B)/Δ) mod 6 )   (max = R)
  = 60 · ( (B−R)/Δ + 2 )       (max = G)
  = 60 · ( (R−G)/Δ + 4 )       (max = B)
H = 0                          (Δ = 0)
```
**丸め（表示）**: H 整数°、S/L 整数%。**逆変換**（編集用）も `core/color/convert.ts` に対で実装。

---

## 4. HSV / HSB カード

**入力**: sRGB（0–1）。**出力**: H[0,360)、S/V[0,100]%。色相 H は HSL と同一。

```
V = max
S = 0           (max = 0)
  = Δ / max     (otherwise)
```
**丸め（表示）**: H 整数°、S/V 整数%。

---

## 5. 相対輝度カード

**目的**: WCAG 相対輝度 Y（カタログ B）。コントラスト計算の基礎値。

```
線形化: R_lin, G_lin, B_lin  （§1.1 gamma decode）
Y = 0.2126·R_lin + 0.7152·G_lin + 0.0722·B_lin     （範囲 0–1）
```
- **丸め（表示）**: 小数4桁。
- 係数は**計算定数**として `contrast.ts` に保持（[06 §3](./06_static_assets_schema.md)：データではない）。
- 出典: WCAG 2.1 「relative luminance」定義。

---

## 6. WCAGコントラスト比カード（＋対白/黒）

**目的**: 前景/背景の可読性判定（カタログ C）。**入力**: 2色（fg/bg）。

```
L1 = max(Y_fg, Y_bg),  L2 = min(Y_fg, Y_bg)
ratio = (L1 + 0.05) / (L2 + 0.05)            （範囲 1.0–21.0）
```
- **丸め（表示）**: 小数2桁、`X.XX : 1` 形式。**判定は丸め前の値**で行う（境界誤判定を防ぐ）。
- **合否判定**: [06 §3](./06_static_assets_schema.md) の `standards/wcag.json` の各 `minRatio` と比較し、AA/AAA・通常/大・UI(3:1) のバッジを出す。
- **大きい文字の定義**: ≥18pt、または ≥14pt 太字（`largeText`）。プレビュー側のサイズと連動（§9）。
- **対白/黒（単色カード）**: 当該色 vs `#ffffff` と vs `#000000` の比を出し、**どちらの地で読みやすいか**を示す。
- 出典: WCAG 2.1 SC 1.4.3 / 1.4.6 / 1.4.11。

---

## 7. CIEDE2000 色差カード（ΔE00）

**目的**: 2色の知覚的な違い（カタログ D）。**入力**: 2色。**経路**: sRGB → 線形RGB → XYZ → CIELAB → ΔE00。

### 7.1 sRGB(線形) → XYZ（D65）
```
[X]   [0.4124564 0.3575761 0.1804375] [R_lin]
[Y] = [0.2126729 0.7151522 0.0721750]·[G_lin]
[Z]   [0.0193339 0.1191920 0.9503041] [B_lin]
```
出典: sRGB→XYZ（D65）標準行列。Y は §5 の相対輝度と一致する。

### 7.2 XYZ → CIELAB（基準白 D65: Xn=0.95047, Yn=1.00000, Zn=1.08883）
```
f(t) = t^(1/3)                       (t > (6/29)^3 = 0.008856)
     = (1/3)(29/6)^2 · t + 4/29       (otherwise)
L* = 116·f(Y/Yn) − 16
a* = 500·(f(X/Xn) − f(Y/Yn))
b* = 200·(f(Y/Yn) − f(Z/Zn))
```

### 7.3 CIEDE2000（kL=kC=kH=1）
標準式（Sharma et al. 2005 の定式）。色1=(L1,a1,b1), 色2=(L2,a2,b2)。
```
C1=√(a1²+b1²), C2=√(a2²+b2²),  C̄=(C1+C2)/2
G = 0.5·(1 − √(C̄⁷/(C̄⁷+25⁷)))
a1'=a1(1+G), a2'=a2(1+G)
C1'=√(a1'²+b1²), C2'=√(a2'²+b2²)
h1'=atan2(b1,a1') [→0,360), h2'=atan2(b2,a2') [→0,360)

ΔL'=L2−L1
ΔC'=C2'−C1'
Δh' = 0                         (C1'·C2'=0)
    = h2'−h1'                   (|h2'−h1'|≤180)
    = h2'−h1'−360               (h2'−h1'>180)
    = h2'−h1'+360               (h2'−h1'<−180)
ΔH' = 2·√(C1'·C2')·sin(Δh'/2)

L̄'=(L1+L2)/2,  C̄'=(C1'+C2')/2
h̄' = h1'+h2'                   (C1'·C2'=0)
   = (h1'+h2')/2                (|h1'−h2'|≤180)
   = (h1'+h2'+360)/2            (|h1'−h2'|>180, h1'+h2'<360)
   = (h1'+h2'−360)/2            (|h1'−h2'|>180, h1'+h2'≥360)

T = 1 − 0.17·cos(h̄'−30) + 0.24·cos(2h̄') + 0.32·cos(3h̄'+6) − 0.20·cos(4h̄'−63)
Δθ = 30·exp(−((h̄'−275)/25)²)
RC = 2·√(C̄'⁷/(C̄'⁷+25⁷))
SL = 1 + (0.015·(L̄'−50)²)/√(20+(L̄'−50)²)
SC = 1 + 0.045·C̄'
SH = 1 + 0.015·C̄'·T
RT = −sin(2·Δθ)·RC

ΔE00 = √( (ΔL'/SL)² + (ΔC'/SC)² + (ΔH'/SH)² + RT·(ΔC'/SC)·(ΔH'/SH) )
```
- 三角関数は度数法で扱う（実装時は度↔ラジアン変換に注意）。
- **丸め（表示）**: 小数2桁。
- **読み方の目安**（表示補助）: ΔE00 < 1 ≈ ほぼ識別不能 / 1–2 ≈ 訓練された目で識別 / 2–10 ≈ 一見して違う / >10 ≈ 明確に別色。

## 8. パレットへの拡張（先回りメモ）

ペアの色差・コントラストは、パレットモードで**全ペアのマトリクス**に拡張する（カタログ C/D の★）。
スプリント1ではペア単位を確定し、マトリクスは同じ `core/color` 関数を N×N に適用するだけにする（計算の再利用）。

---

## 9. テキスト可読性プレビューカード

**目的**: 数値でなく**文脈での見え方**を確認（検証と設計の橋渡し、カタログ J）。計算ではなく描画仕様。

- fg/bg を使い、サンプルを描画: 本文（通常）、見出し（大）、太字小、リンク等。
- 各サンプルに **WCAG 判定バッジ**（§6 の結果）を併記。サンプルの文字サイズは大/通常の閾値（≥18pt / ≥14pt 太字）と連動。
- 編集時はリアルタイム更新（[03 §1](./03_ui_skeleton.md)）。

## 10. 検証用参照値（テストデータ）

`core/color` のユニットテスト基準（[05 §5](./05_data_model_and_card_contract.md) のテスト可能性）。

### コントラスト比
| fg | bg | 期待 ratio | 判定 |
|----|----|-----------|------|
| #000000 | #ffffff | 21.00 | AAA 合格 |
| #ffffff | #ffffff | 1.00 | 不合格 |
| #777777 | #ffffff | 4.48 | AA大のみ |
| #767676 | #ffffff | 4.54 | AA通常合格 |

### 相対輝度
| 色 | 期待 Y |
|----|--------|
| #000000 | 0.0000 |
| #ffffff | 1.0000 |
| #808080 | 0.2159 |

### CIEDE2000
- 実装は Sharma et al. (2005) の **34組の標準テストペア**で照合する（論文付随データを出典として参照）。
- 代表例（目安）: 白(#fff) vs 黒(#000) は大きな ΔE00、近接グレー同士は小さな ΔE00 になることを確認。

> 数値は実装時に各式で再算出して固定する。表は丸め2桁の目安。
