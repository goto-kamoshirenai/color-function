# 13. コミット規約

> ステータス: 確定 / 最終更新: 2026-06-10
> 関連: [TDD 開発手順書](./12_tdd_workflow.md) / [AGENTS.md](../AGENTS.md)

デファクトスタンダードである **[Conventional Commits](https://www.conventionalcommits.org/)** に準拠した、簡単なコミット規約。
本リポジトリの全コミットはこの規約に従う。

---

## 1. 形式

```
<type>(<scope>): <要約>

<本文（任意）>

<フッタ（任意）>
```

- 1行目（ヘッダ）のみ必須。`<scope>` は任意。
- **要約**: 簡潔に「何をしたか」。日本語可・現在形・句点なし・50字程度まで。
- **本文**: なぜ／何を。72字前後で折り返す。複数行可。
- **フッタ**: `Co-Authored-By:` 等。

---

## 2. type（必須）

| type       | 用途                                                 |
| ---------- | ---------------------------------------------------- |
| `feat`     | 機能の追加                                           |
| `fix`      | バグ修正                                             |
| `docs`     | ドキュメントのみの変更                               |
| `test`     | テストの追加・修正（実装変更を伴わない）             |
| `refactor` | 挙動を変えないコード整理                             |
| `style`    | 整形のみ（空白・セミコロン等。挙動・意味に影響なし） |
| `perf`     | パフォーマンス改善                                   |
| `chore`    | ビルド・依存・設定など雑務                           |
| `build`    | ビルドシステム・依存関係                             |
| `ci`       | CI 設定                                              |

> 破壊的変更は `feat!:` のように `!` を付けるか、フッタに `BREAKING CHANGE:` を記す。

---

## 3. scope（任意）

変更範囲を示すモジュール名。本プロジェクトの主な scope:

`core-color` / `store` / `cards` / `components` / `lib` / `assets` / `app` / `config`

---

## 4. 粒度

- **作業は常に適切な粒度でコミットする** — 1コミット＝**1つの論理的変更**。
- 無関係な変更を1コミットに混ぜない。大きな変更は意味のある単位に分割する。
- TDD では「失敗テスト＋それを通す実装」を**1コミットにまとめてよい**（Red 単独コミットはしない。[12 §6](./12_tdd_workflow.md)）。

---

## 5. push 禁止

- **push は禁止。コミットまでで止める**。リモートへの反映は明示の指示があるまで行わない。

---

## 6. 例

```
feat(core-color): WCAGコントラスト比と相対輝度を実装

CIEDE2000 と共通の sRGB→線形RGB 変換を用意。
参照値（黒×白=21.00 等）でテスト済み。

Co-Authored-By: ...
```

```
test(store): apply の add/remove/reorder を追加
```

```
docs: コミット規約(13)を追加
```

```
fix(cards): ペアモードで FG/BG 入替が反映されない不具合を修正
```

> 旧コミット（`ADD`/`FIX` 等の大文字prefix）は履歴として残すが、**以後は本規約に統一**する。
